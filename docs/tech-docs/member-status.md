# 멤버 상태 관리 기능 (활동/졸업)

이 문서는 멤버의 활동 상태(활동 중/졸업)를 관리하고 졸업한 멤버를 스케줄표에서 자동으로 처리하는 기능에 대해 설명합니다.

## 1. 개요
멤버가 졸업했을 때 데이터를 바로 삭제하지 않고, '졸업' 상태와 '졸업일'을 지정하여 과거 스케줄 기록은 보존하면서 현재 및 미래 스케줄표에서는 자동으로 숨겨지도록 합니다.

## 2. 데이터베이스 사양
`characters` 테이블에 다음 컬럼이 추가되었습니다.

| 컬럼명 | 타입 | 기본값 | 설명 |
| :--- | :--- | :--- | :--- |
| `status` | `text` | `'active'` | 멤버의 현재 상태 (`'active'`, `'graduated'`) |
| `graduation_date` | `date` | `NULL` | 멤버의 공식 졸업일 |

## 3. 필터링 로직
`getScheduleFromSupabase` 유틸리티 함수에서 다음 로직에 따라 멤버를 필터링합니다.

- **표시 조건**:
  - `status`가 `'active'`인 경우
  - `status`가 `'graduated'`이지만, 지정된 `graduation_date`가 현재 조회 중인 주의 **월요일보다 크거나 같은 경우**
- **숨김 조건**:
  - `status`가 `'graduated'`이고, `graduation_date`가 현재 조회 중인 주의 **월요일보다 이전인 경우**

## 4. 관리자 기능 (Admin UI)
- **멤버 정보 수정**: 관리자 페이지 사이드바의 멤버 목록 옆 ✏️ 버튼을 통해 접근 가능합니다.
- **졸업 설정**: 수정 모달 내 "멤버 상태 관리" 섹션에서 상태를 '졸업'으로 변경하고 졸업일을 지정할 수 있습니다.
- **데이터 보존**: 졸업한 멤버는 필터링되어 스케줄표에 나타나지 않을 뿐, 데이터베이스에는 그대로 유지되므로 어드민 관리자 화면에서는 여전히 확인 및 수정이 가능합니다.

## 5. 참고 사항
- 일반 멤버는 자신의 프로필(이름, 이미지, 정기 휴방 등)만 수정할 수 있으며, 졸업 상태 정보도 확인할 수 있습니다.
- 졸업 예정일이 미래인 경우, 해당 날짜가 도래하기 전까지는 스케줄표에 정상적으로 노출됩니다.
