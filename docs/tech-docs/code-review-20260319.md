# Hanavi Schedule - 프로젝트 코드 리뷰 결과 (2026-03-19)

이 문서는 프로젝트 전체의 **보안(Security)** 및 **사용자 편의성(UI/UX)** 관점에서의 리뷰 결과와, 이에 따른 조치 내역을 기록한 문서입니다.

---

## 🛡️ 1. 보안(Security) 관점 리뷰 및 조치 내역

### 🚨 주요 보안 취약점 (Critical)
**1. Characters 테이블 RLS (Row Level Security) 정책 개방**
- **발견 내용:** `Public Write Characters` 정책이 `using (true) with check (true)`로 설정되어 있어 누구나 Characters 테이블의 데이터를 삽입, 수정, 삭제할 수 있는 상태였습니다.
- **조치 결과:** `using (public.is_admin())` 정책이 적용된 신규 SQL 마이그레이션(`20260319_secure_characters_rls.sql`)을 만들어 관리자만 쓰기/수정이 가능하도록 접근을 차단했습니다.

### ⚠️ 잠재적 보안 이슈 (Warning)
**2. XSS (Cross-Site Scripting) 취약점 가능성**
- **발견 내용:** `src/components/ScheduleCell.tsx`에서 스케줄 컨텐츠 렌더링 시 `dangerouslySetInnerHTML`를 사용하여, RLS 취약점과 결합될 시 악성 스크립트 실행이 가능한 상태였습니다.
- **조치 결과:** 렌더링 직전 HTML 문자열을 안전하게 정제(Sanitize)하도록 `isomorphic-dompurify` 라이브러리를 도입하여 잠재적 공격을 무력화했습니다.

**3. 미사용/중복 로그인 API 라우트 존재**
- **발견 내용:** 프론트엔드(`admin/page.tsx`)에서 자체적으로 Supabase 인증을 수행하고 있음에도, `api/admin/login` 엔드포인트가 환경 변수를 검증하는 가짜 껍데기 형태로 남아 보안 및 무차별 대입 공격 위협이 있었습니다.
- **조치 결과:** 사용되지 않는 API 디렉터리(`src/app/api/admin/login`) 자체를 완전히 삭제하여 공격 표면을 없앴습니다.

## 🎨 2. 사용자 편의성(UI/UX) 관점 리뷰

### 🌟 훌륭한 UI/UX 요소 (Good Points)
1. **다중 내보내기 & 캘린더 연동 최적화:** ICS 파일 다운로드 및 `navigator.share`를 활용한 PNG 내보내기는 모바일 친화적인 최고의 편의성을 갖추고 있습니다.
2. **PWA & 햅틱 피드백:** 웹 앱 기능(`usePWA`) 및 선택 시 미세한 진동(`web-haptics`)은 사용자 경험을 한 단계 끌어올리는 네이티브 급 기술입니다.
3. **가변 폰트 크기 시스템:** `complexityScore` 로직을 통해 텍스트 길이나 줄 바꿈에 따라 폰트 크기가 조정되어 작은 셀에서도 깔끔한 레이아웃이 유지됩니다.
4. **YouTube 최신 방송 자동 연결 (Auto Link):** 관리자의 운영 편의성을 위해 영상 링크를 자동으로 가져와 채워주는 강력한 기능을 제공하고 있습니다.

### 💡 남은 개선 과제 (Backlog에 등록됨)
**1. 관리자 페이지 에러 처리 통일 (Alert vs Toast)**
- 문제점: 현재 관리자 페이지의 알람, 오류 처리가 기본 `alert`로 되어 있습니다.
- 개선계획: 사용자 페이지와 동일하게 `sonner` Toast UI를 도입하여 통일감 있고 깔끔한 관리자 피드백 환경을 구축합니다.

**2. 하나비 단체 방송 셀 병합 기능 기획**
- 문제점: 하나비 그룹 단체 방송 시 6개의 독립된 셀이 반복되어 표시되어 가독성이 저하됩니다.
- 개선계획: 스케줄표에서 단체 방송(합방) 속성이 감지되면 시각적으로 독립된 6개 셀이 아닌 1개의 거대한 통합 셀로 표시하는 뷰 로직 개선이 필요합니다.
