# Release Notes - v0.2.0

## 개요
v0.2.0 버전에서는 멤버 관리 시스템의 유연성을 확보하고, 보안 강화 및 실사용 편의를 위한 다양한 기능 개선이 이루어졌습니다.

## 주요 변경 사항

### 1. 멤버 관리 및 필터링 고도화
- **멤버 상태 관리**: 멤버의 활동/졸업 여부를 설정할 수 있는 필드가 추가되었습니다.
- **졸업일 기준 필터링**: 졸업한 멤버는 기본 스케줄표에서 제외되지만, 과거 스케줄 기록에는 유지되도록 로직을 개선하였습니다.
- **커스텀 다시보기 텍스트**: 멤버별로 고유한 '다시보기' 텍스트를 설정할 수 있습니다. (예: 체리 - '다시보고팡', 엘라 - '다시봐라')

### 2. 새로운 링크 및 데이터 연동
- **씨미(Cime) 전용 링크**: 멤버 프로필 내 씨미 전용 링크 필드를 추가하여 직접 연결이 가능합니다.
- **데이터 안정화**: 입력 필드에서 발생하던 데이터 반영 오류 및 `graduation_date` 미기입 시의 예외 처리를 완료하였습니다.

### 3. 보안 및 아키텍처 개선
- **RLS(Row Level Security) 정책 강화**: Supabase 보안 취약점을 해결하고 API 권한 제어를 통해 무분별한 데이터 변경을 방지합니다.
- **코드 리팩토링**: `ScheduleGrid` 및 하드코딩된 데이터들을 정리하고 DB 연동 구조로 전환하여 유지보수성을 높였습니다.

### 4. UI/UX 및 아키텍처 고도화
- **Skeleton UI 도입**: 데이터 로딩 중 레이아웃 흔들림(Layout Shift) 방지를 위해 `ScheduleSkeleton` 컴포넌트를 적용했습니다.
- **데이터 페칭 로직 통합**: `useSchedule` 훅을 통해 메인/관리자 페이지의 데이터 요청을 일원화하고 SWR 캐시를 공유하도록 개선했습니다.
- **관리자 경험 개선**: 멤버 추가/삭제 시 페이지 전체를 새로고침 없이 즉각적으로 캐시가 갱신되도록 최적화했습니다.

### 5. 문서 및 리서치
- 사용자 가이드 및 릴리즈 문서를 `docs/` 하위 체계로 통합하였습니다.
- 프로젝트 전반의 기술적 상태를 점검한 [코드 리뷰 보고서](../tech-docs/code-review-20260315.md)를 작성했습니다.

## 향후 계획 (v0.3.0)
- **Toast UI 도입**: 로직 처리 결과에 대한 부드러운 알림 피드백
- **Error Boundary 설정**: 예외 상황 발생 시 앱 안정성 확보
- **YouTube API 최적화**: 서버 측 캐싱 강화
- **다크 모드 고도화**: 테마 스위처 도입 및 심미성 개선
